Project Overview

This Project is done only to Showcase the API integration, Celery implementation, Payment Gateway integration.
For API there are Three main APIs that has been used in this Project, These are:
Media Stack for fetching News Details,
Google ReCaptcha for Captcdha Verification,
RazorPay For Payment Gateway.
For Celery Implementation Render's Redis instance has been used
All the Activities has been completed only using the free tier versions that's the main reason why I have used Media Stack over News API since news API does'nt provide news on countries other than US and I Personally wanted to make a News Filter Option, For that More than One country's News and Languages were Proving to be Mandatory Requirements
Also For the Subscription there are basically two plans, one is monthly and another is yearly, a Scheduler is used to constantly check for subscription validity every mid night so as to expire the subscription when the right time comes.
Additionally after Deploying I found out that Render makes the website inactive if it does'nt fount any activity for 15minutes and sends the website to sleep mode, which is kind of annoying since it takes a lot of time to open from sleep mode so i created a bash file naming setup.sh to run all the celery and web commands, and also created another task which will ping the website after every 5minutes, thus the website can't get into sleep mode, making my website run faster!

Project Overview

This Project mainly focuses on the Data base concept of Django. All the CRUD functions have been successfully implemented. Basically 4 Tables have been Used in this project.
First is the by default User model in Django.
Second is the Quote Model that has been created keeping an attribute user as the Foreign key for User Model. This model basically stores all the necessary attributes that is required to create/edit a Quote for Example: Font name, Quote Body, Copyright name, Caption etc etc
Third a table for storing the Likes and the user as foreign Key of User Model to know which user Liked and another attribute quote, a Foreign Key for Quote Model to know which Quote the user Liked
Similarly a Comment Model is Created just Like the Like, this is for storing the comments.
